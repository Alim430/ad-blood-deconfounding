# Peripheral blood RNA in Alzheimer's disease is predominantly myeloid composition

Analysis code and reproducibility materials for the manuscript:

> **Blood gene-expression biomarkers of Alzheimer's disease mainly reflect immune-cell composition, not disease-specific gene activity: a multi-modal audit.**
> Alimujiang Tudiyusufu.  Contact: Alim_T@foxmail.com

A multi-modal audit (bulk transcriptomics, single-cell RNA-seq, a deep generative
embedding, and the single-cell TCR repertoire; ~1,350 participants) testing whether the
Alzheimer's disease (AD) signal in peripheral-blood RNA is primarily a **cell-composition
phenotype** rather than a cell-intrinsic transcriptional one.

## Key findings
- **~67%** of "robust" blood-RNA biomarkers do not survive de-confounding for leukocyte composition.
- In independent data (ADNI), the survivors **collapse to the genome-wide background** under symmetric composition adjustment (direction-concordance **88% → 48%**), ≈40× greater than random gene sets, reproduced by a minimal two-marker model.
- A formal equivalence test (TOST) confirms negligible residual effect sizes.
- Donor-level single-cell analysis recovers **≈6** within-cell-type genes vs **873** from a pseudoreplicated cell-level test (genomic inflation λ ≤ 1).
- A classifier on **six composition scores alone** matches the de-confounded transcriptomic panel (**AUC 0.627 vs 0.63**).
- The central monocyte shift is reproduced across **three** deconvolution methods (Danaher, Bisque, MuSiC).

## Reproduce
1. **Data access:** see [`docs/data_access.md`](docs/data_access.md). All datasets are public; ADNI is controlled-access (apply via adni.loni.usc.edu). **No raw data is redistributed in this repository.**
2. **Environment:**
   - Python: `pip install -r requirements.txt`
   - R: package versions in `sessionInfo.txt`; key packages listed in `docs/data_access.md`.
3. **Run** the scripts in `script/` in numbered order (`Paper1_Step2_*` → `Paper1_Step33b_*`). Each step reads from / writes to `results/`. Large intermediate objects (`*.RData`, `*.rds`) are **regenerated by the scripts**, not stored here.

## Repository layout
```
script/                       analysis scripts (R + Python), numbered by step
script/ai/                    optional / server-only analyses used in v7 (ML, scVI, pseudobulk)
docs/
  ├─ data_access.md                     how to obtain each public dataset (+ ADNI DUA)
  ├─ Paper1_Supplementary_Materials.md  supplementary methods + 21 robustness analyses (S1–S21)
  ├─ PIPELINE.md                        workflow overview
  └─ SINGLE_CELL_PIPELINE.md            single-cell sub-pipeline overview
figures/pub/                  publication figures (PDF + PNG)
results/                      (generated locally by the scripts; not tracked in this repository)
```
The manuscript itself is distributed through the journal / preprint, not this repository,
which contains the analysis code and reproducibility materials only.

## What is NOT in this repository
Raw GEO expression matrices, any ADNI data (controlled-access, redistribution prohibited),
derived result tables, and large intermediate `.RData`/`.rds` files are **excluded by design**
(see `.gitignore`). The scripts download or point to the original public sources and regenerate
the result tables locally.
